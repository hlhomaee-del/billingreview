Section 2

Meet the 3 Code Sets

In medical billing and coding, three main "books" (code sets) are used to report what happened during a patient encounter. You need to know what each one is for — and when to use it.

📖What CPT, HCPCS Level II, and ICD-10-CM each do
🔄How they are different from each other
📝When each one is used in practice
📌 What to Do Click each code set card below to expand its details. Then use the comparison table. Finally, complete the mini-scenarios to check your understanding.
CPT
Current Procedural Terminology
Click to expand ▼
🔧What it reports: Procedures and services performed by healthcare providers — things the provider did to the patient.
📋Code format: 5-digit numeric codes (e.g., 99213, 27447)
🏥Examples: Office visits, surgeries, lab tests ordered by the provider, physical therapy treatments, injections
🎯Think of it as: "What did the provider DO?"
📚Published by: American Medical Association (AMA)
Example: A patient comes in for a routine office visit. The provider examines the patient and reviews their medication. That visit → CPT code 99213 (Office/outpatient visit, established patient, moderate complexity).
HCPCS II
Healthcare Common Procedure Coding System Level II
Click to expand ▼
🩹What it reports: Supplies, equipment, medications, and services not covered in CPT — especially those billed to Medicare/Medicaid.
📋Code format: 1 letter + 4 numbers (e.g., A6216, E0601, J0696)
🏥Examples: Durable medical equipment (wheelchairs, walkers), wound dressings, ambulance services, injectable drugs, orthotic/prosthetic devices
🎯Think of it as: "What did the patient RECEIVE or what was SUPPLIED?"
📚Published by: Centers for Medicare & Medicaid Services (CMS)
Example: A patient is prescribed a wheelchair after hip surgery. The supplier bills for that wheelchair → HCPCS Level II code K0001 (Standard manual wheelchair).
ICD-10-CM
International Classification of Diseases, 10th Revision
Click to expand ▼
🩺What it reports: Diagnoses, conditions, diseases, symptoms, and reasons for the encounter — the why of the visit.
📋Code format: 1 letter + 2 numbers + decimal + up to 4 more characters (e.g., J06.9, M54.5, Z00.00)
🏥Examples: Type 2 diabetes, hypertension, acute sinusitis, broken wrist, routine wellness exam
🎯Think of it as: "WHY did the patient come in? What is WRONG?"
📚Published by: World Health Organization (WHO) / adapted by CMS for US use
Example: That same office visit patient has Type 2 diabetes. The biller needs to report the diagnosis → ICD-10-CM code E11.9 (Type 2 diabetes mellitus without complications).

SOAP Notes Interactive Lab

Before a coder can assign any code, they must read the provider's documentation. The most common format for that documentation is the SOAP note. You need to understand exactly what belongs in each part.

📝What each letter in SOAP stands for
🔍What information belongs in each section
🎯How to identify SOAP sections in real notes
📌 What to Do First, click each SOAP card to learn what it means. Then drag the note fragments into the correct SOAP column. Finally, answer the concept check questions.
S

Subjective

What the PATIENT says. These are the patient's own words — their complaints, symptoms, and history as they report them.

Examples: "Chief complaint: headache for 3 days." / "Patient reports nausea and dizziness." / "Denies fever or chills."

⚡ Coder's eye: This section helps identify the reason for the visit.

O

Objective

What the PROVIDER observes and measures. These are measurable, observable findings — not the patient's opinion.

Examples: Vital signs (BP 138/84), physical exam findings, lab results, imaging results, temperature.

⚡ Coder's eye: This section supports the diagnosis with documented evidence.

A

Assessment

The provider's diagnosis or clinical impression. This is what the provider concludes after reviewing subjective and objective information.

Examples: "Diagnosis: Acute sinusitis." / "Assessment: Possible anxiety disorder." / "Impression: Type 2 diabetes, uncontrolled."

⚡ Coder's eye: This is the most important section for ICD-10-CM coding.

P

Plan

What the provider is going to DO next. This includes treatments, prescriptions, referrals, tests ordered, and follow-up instructions.

Examples: "Prescribe amoxicillin 500mg TID x10 days." / "Order CBC and CMP." / "Refer to cardiologist." / "Return in 2 weeks."

⚡ Coder's eye: Procedures and services coded in CPT often come from this section.

Section 4

What Coding Is

Medical coding is the process of translating clinical documentation into standardized alphanumeric codes. Understanding why this exists — and why precision matters — is foundational to everything else you'll learn.

💡What coding means in practice
📤How documentation becomes a claim
⚠️Why vague documentation causes problems
📌 What to Do Click through the flow below to see how a patient visit becomes a coded claim. Then answer the questions that follow.

The Coding Flow: Visit → Documentation → Code → Claim

Step 1: Patient Visit

A patient comes in with a complaint. The provider examines them, makes decisions, and provides services. Click to see what happens next.

↓
Step 2: Provider Documents Everything

The provider writes a SOAP note or other clinical documentation. Every detail they document either supports or limits what can be coded. If it isn't documented, it cannot be coded.

↓
Step 3: Coder Reviews Documentation

A medical coder reads the documentation and translates it into standardized codes. They use ICD-10-CM for diagnoses, CPT for procedures, and HCPCS Level II for supplies/equipment.

⚠️ Coders do not guess. They only code what is documented. Assuming or guessing at diagnoses is a compliance violation.
↓
Step 4: Codes Go on the Claim

The codes are placed on a claim form (like a CMS-1500 for outpatient) and submitted to insurance. The insurer uses these codes to determine: Is this covered? How much do we pay?

↓
Step 5: The Claim is Adjudicated (Paid, Denied, or Audited)

If codes are wrong, vague, or unsupported by documentation: the claim may be denied, underpaid, delayed, or flagged for audit. This costs the practice money and the patient confusion.

✅ Accurate, specific coding = clean claims = faster payment.

Why Specificity Matters — Two Coders, One Patient

❌ Vague Documentation
Assessment: leg pain

A coder reading this note cannot determine: Which leg? What kind of pain? Is it a muscle, nerve, bone, or joint issue? The coder is stuck. The code assigned will be nonspecific, which may result in claim denial.

✅ Specific Documentation
Assessment: Acute right calf muscle strain, first episode

Now the coder knows: the side (right), the structure (calf muscle), the type (strain), and the episode (first). This maps directly to a specific ICD-10-CM code — a clean, payable claim.

The Rules of ICD-10-CM Code Structure

Character 1 — Always a Letter

The first character is always a letter (A through Z). It identifies the chapter of the ICD-10-CM codebook where this code lives.

Example: S = Injury codes | E = Endocrine/metabolic | J = Respiratory | M = Musculoskeletal
Characters 2–3 — Numbers

Characters 2 and 3 are usually numbers. Together with Character 1, they complete the category code — a 3-character code representing a broad condition.

S52 = Fracture of forearm (the broad category)
Characters 4–7 — Specificity

Characters after the decimal point (positions 4–7) add clinical detail: which bone, which side, which type of fracture, which encounter, etc.

.201A = Shaft of right ulna, unspecified fracture, initial encounter
⚠️ Remember: The category code (first 3 characters) is the starting point, but it is rarely specific enough for billing. You almost always need to use the most specific code supported by the documentation.

Rapid Review

A quick-reference summary of everything covered in this lab. Use this to review before a test or after you finish a section.

📚 The 3 Code Sets — Quick Reference
CPT — Reports procedures and services. 5-digit numeric codes. Published by AMA. Ask: "What did the provider DO?"
HCPCS Level II — Reports supplies, DME, equipment, injectable drugs, ambulance. 1 letter + 4 numbers. Published by CMS. Ask: "What was SUPPLIED?"
ICD-10-CM — Reports diagnoses and conditions. Starts with a letter. Published by WHO/CMS. Ask: "WHY did the patient come in?"
📋 SOAP Notes — Definitions
S — Subjective: What the patient says. Complaints, symptoms, history in the patient's own words.
O — Objective: What the provider observes and measures. Vital signs, physical exam, lab results.
A — Assessment: The provider's diagnosis or clinical impression. This is where ICD-10-CM codes come from.
P — Plan: What the provider orders. Prescriptions, referrals, tests, follow-up. CPT codes often come from here.
🔡 ICD-10-CM Code Structure
Character 1: Always a letter (A–Z). Identifies the chapter/body system.
Characters 1–3: Together = the category code. Broad condition grouping.
Characters 4–7 (after decimal): Add specificity — site, side, type, encounter.
Rule: Always use the most specific code supported by documentation.
🎯 Category Codes & Specificity
A category code = exactly 3 characters (e.g., J18, E11, M54)
More characters = more specific = better documentation support
E11 = Type 2 diabetes (broad category)
E11.65 = Type 2 diabetes WITH hyperglycemia (specific complication added)
Specificity comes from documentation — not guesswork.
⚠️ Common Mistakes to Avoid
❌ Using CPT to report a diagnosis (CPT = procedures only)
❌ Using ICD-10-CM to report a service (ICD-10-CM = diagnoses only)
❌ Coding a symptom instead of the diagnosis when the diagnosis IS documented
❌ Guessing the causative organism when none is documented
❌ Using a category code when a more specific code is available and documented
❌ Adding information the provider did not document
⚡ If You See This, Think This
"What did the provider do?" → CPT
"What diagnosis was documented?" → ICD-10-CM
"DME, walker, wheelchair, ambulance" → HCPCS Level II
"Patient says, patient reports, patient denies" → Subjective (S)
"BP, temp, exam findings, lab values" → Objective (O)
"Diagnosis, impression, assessment" → Assessment (A) → ICD-10-CM
"Prescribe, refer, order, follow up" → Plan (P) → CPT/HCPCS
"First 3 characters of ICD-10-CM code" → Category code
"More characters = ?" → More specificity
"Documentation is vague" → Code what IS documented; do not guess
💡 Coding Fundamentals — 5 Rules
1. Code only what is documented.
2. Use the most specific code supported by the documentation.
3. Don't code a symptom if the diagnosis is documented.
4. ICD-10-CM is used on almost every claim — it explains WHY services were provided.
5. If the documentation is unclear, query the provider — do not assume.
